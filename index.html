noTouchi

jQuery plugin to stop customers from changing sensitive software settings

How many times have you got this call...

Yeah... I wasn't thinking and I changed some important settings. My software isn't working now.

Why did you do that?

I don't know.

Didn't we ask you NOT to touch those settings?

Yes.

Here's a solution

With one line of jQuery and one line of PHP... you can protect sensitive areas in your web applications.

noTouchi assures you that sensitive changes will only be made with the help of a technical support person.

How it works

noTouchi will generate a random four digit support code. The technician then returns a four digit key based on these rules:

  • first digit: the first number
  • remaining digits: multiply the last number by 100
For instance, if the support code is the support key would be
  • 2 is the first number
  • 8 x 100 = 800

These are just the default settings. You can build your own custom version of the key validation rules.

Try it out

Click the button below. Once you provide the correct key, the PHP component will create a 5 minute support session on the server... allowing you to make necessary changes.

How to use

Insert one line of jQuery into your ajax success event:

if ( $("body").noTouchi({"response" : msg}) != true ) return false;

Insert one line into your PHP file above the content you want to protect:

require("noTouchi.php");

Simple Example

Super easy... with just one line of code

$.ajax({
	type: "POST",
	url: "your.php",
	success: function(msg){

		//evaluate the msg for a noTouchi response
		if ( $("body").noTouchi({"response" : msg}) != true ) return false;
		
		//your code to execute once a session is created
	}
});

Long Example

Options allow you to supply a custom php script name/location

$.ajax({
	type: "POST",
	url: "your.php",
	success: function(msg){

		//evaluate the msg for a noTouchi response
		//you can even specify a custom location/name for the noTouchi php component
		var a = $("body").noTouchi(
			{
				"php_postTo" : "./yourFolder/yourFile.php",
				"response" : msg
			}
		);

		if ( a != true ) return false;
		
		//your code to execute once a session is created
	}
});
